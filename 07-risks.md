# 7. 리스크 및 대응 방안

| 리스크 | 영향 | 대응 |
|--------|------|------|
| TCGPlayer 크롤링 구조 변경 | 해외판 카드 시세 수집 불가 | PriceCharting + eBay 폴백 + 구조 변경 감지 알림 |
| **eBay API 키 만료/제한** | 영어판 실거래 데이터 수집 불가 | Sold listings 크롤링으로 폴백. API 갱신 주기 설정 파일로 관리 |
| **번개장터 크롤링 차단** | 국내 실거래 매물 가격 수집 불가 | 자동 스킵 → 네이버 카페(공개)/구글시트 폴백. 3일 연속 차단 시 해당 소스 자동 비활성화 |
| **당근마켓 크롤링 차단** | 국내 실거래 매물 가격 수집 불가 | 자동 스킵 → 번개장터/네이버 카페 폴백. 당근은 차단 강도가 가장 높아 가장 먼저 비활성화될 가능성 |
| **번개장터/당근 ToS 위반 경고** | 계정 차단, 법적 통지 가능성 | 즉시 해당 크롤러 비활성화. 구글시트 수동 입력으로 전환 |
| **번개장터 수수료 변동** | 표시가격과 실거래가 괴리 확대 | UI에 "판매 희망가 (수수료 별도)" 명시. 수수료율 설정 파일로 관리 |
| **네이버 카페 공개 게시판 데이터 부재** | 국내판 시세 수집 불가 (로그인 필요 게시글 접근 불가) | 구글시트 수동 입력 폴백이 실질적 주력 수단. 초기부터 구글시트 병행 운영 권장 |
| 네이버 카페 시세글 형식 변경 | LLM 파싱 정확도 하락 | 서브에이전트 프롬프트 업데이트 + 파싱 실패 로그로 패턴 파악 |
| **BrickEconomy 크롤링 차단** | 레고 중고 시세 수집 불가 | LEGO 공식 정가만 표시 + BrickLink API v2 연동으로 대체 (향후 확장) |
| 그레이딩 가격 데이터 부족 | 특정 등급 시세 공백 (특히 한국판) | 해당 등급 항목을 `graded` 배열에서 제외, UI에 표시 생략 |
| 한국판↔해외판 카드 매핑 실패 | 동일 카드인데 별도 항목으로 표시 | LLM 기반 매핑 + 수동 매핑 테이블 보정 |
| **일본판(jp) 수집 방법 미확인** | 일본판 시세 추적 불가 | v1에서 완전 제외. v2에서 Mercari Japan/Yahoo Japan 수집 방법 조사 후 추가 |
| GitHub Actions 무료 한도 초과 | 자동 수집 중단 | 수집 주기 조절 (주 3회로 감소) 또는 국내 플랫폼 크롤링 비활성화 |
| **한국은행 API 장애** | 원화 가격 부정확 | 최근 유효 환율 캐시 (`exchange_rate.json`) 사용. 최대 7일 캐시 허용 |
| 카드/레고 목록 변경 | 새 상품 추적 누락 | `watchlist.json` KYH가 직접 수정 후 push |
