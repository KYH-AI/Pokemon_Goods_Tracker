# 2. 워크플로우 정의

## 2.1 전체 흐름 개요

```
┌─────────────────────────────────────────────────────────┐
│                   GitHub Actions Cron                    │
│              (하루 2회: 09:00 KST, 18:00 KST)            │
└──────────────────────┬──────────────────────────────────┘
                       │
         ┌─────────────▼─────────────┐
         │   Step 1: 데이터 수집      │
         │   (각 소스별 스크립트)      │
         └─────────────┬─────────────┘
                       │
         ┌─────────────▼─────────────┐
         │   Step 2: 데이터 정제      │
         │   (정규화, 매핑, 병합)      │
         └─────────────┬─────────────┘
                       │
         ┌─────────────▼─────────────┐
         │   Step 3: 검증            │
         │   (스키마, 데이터 무결성)   │
         └─────────────┬─────────────┘
                       │
         ┌─────────────▼─────────────┐
         │   Step 4: 사이트 빌드      │
         │   (JSON → HTML 생성)       │
         └─────────────┬─────────────┘
                       │
         ┌─────────────▼─────────────┐
         │   Step 5: 배포            │
         │   (gh-pages 브랜치 푸시)   │
         └─────────────┘─────────────┘
```

## 2.2 단계별 상세

### Step 1: 데이터 수집

| 하위 작업 | 처리 방식 | 입력 | 출력 | 비고 |
|-----------|----------|------|------|------|
| 1-1. 해외판 카드 시세 수집 | 스크립트 | `watchlist.json` (카드 목록) | `raw/cards_tcgplayer.json`, `raw/cards_pricecharting.json` | TCGPlayer → PriceCharting 순서, 그레이딩 등급별 가격 포함 |
| 1-2. eBay 실거래 시세 수집 | 스크립트 | `watchlist.json` (카드 목록) | `raw/cards_ebay.json` | Browse API 또는 Sold listings 크롤링. 그레이딩 등급별 낙찰가 |
| 1-3. 국내판 카드 시세 수집 (카페) | 스크립트 | `watchlist.json` (카드 목록) | `raw/cards_kr_naver.json` | 네이버 카페 시세글 크롤링 (폴백: 구글시트) |
| 1-4. 국내 실거래 시세 수집 | 스크립트 | `watchlist.json` (카드 목록) | `raw/cards_bunjang.json`, `raw/cards_daangn.json` | 번개장터 → 당근마켓 순서. 차단 시 자동 스킵 |
| 1-5. 레고 가격 수집 | 스크립트 | `watchlist.json` (레고 목록) | `raw/lego_bricklink.json`, `raw/lego_brickeconomy.json`, `raw/lego_official.json` | BrickLink API → BrickEconomy 크롤링 → 공식가 |
| 1-6. 이벤트 수집 | 스크립트 | 대상 URL 목록 (설정) | `raw/events_official.json`, `raw/events_community.json` | 포켓몬코리아 → 네이버카페 순서 |

**분기 조건**:
- 특정 소스 크롤링 실패 시 → 해당 소스만 스킵, 나머지 계속 진행
- **당근/번개장터 차단 감지 시** → 즉시 해당 소스 스킵 (403/429 응답 또는 CAPTCHA 감지), 로그에 차단 사유 기록
- 전체 소스 실패 시 → 이전 데이터 유지, 에러 로그 기록

**국내 플랫폼 크롤링 전략 (1-4 상세)**:

| 항목 | 번개장터 | 당근마켓 |
|------|---------|---------|
| **진입점** | `m.bunjang.co.kr/search/products?q={keyword}` | `www.daangn.com/hot_articles` + 검색 |
| **수집 대상** | 검색 결과 매물 (제목, 가격, 상태) | 인기매물 + 키워드 검색 결과 |
| **요청 간격** | 3~5초 랜덤 딜레이 | 5~8초 랜덤 딜레이 (차단 강도 높음) |
| **User-Agent** | 모바일 브라우저 UA 로테이션 | 모바일 브라우저 UA 로테이션 |
| **세션 관리** | 쿠키 기반 세션 유지 | 쿠키 없이 stateless 요청 |
| **차단 감지** | HTTP 403, 빈 응답, CAPTCHA 페이지 | HTTP 403, 429, 리다이렉트 |
| **차단 시 행동** | 즉시 중단 + 해당 수집 주기 스킵 | 즉시 중단 + 해당 수집 주기 스킵 |
| **데이터 특성** | 판매 희망가 (수수료 6%+ 별도) | 판매 희망가 (직거래 기반) |

**성공 기준**: 최소 1개 이상의 소스에서 데이터 수집 성공
**검증 방법**: 규칙 기반 — 각 raw JSON 파일 존재 여부 + 비어있지 않은지 확인
**실패 처리**: 스킵 + 로그 (개별 소스 실패) / 에스컬레이션 (전체 실패 → GitHub Actions 알림)

### Step 2: 데이터 정제

| 하위 작업 | 처리 방식 | 설명 |
|-----------|----------|------|
| 2-1. 카드명 매핑 | 스크립트 + LLM 판단 | 영문 카드명 → 한국어 카드명 매핑 (매핑 테이블 기반, 미매핑 시 LLM 추론) |
| 2-2. 에디션 분류 | 스크립트 | 소스별 카드를 에디션(`kr`/`en`/`jp`)으로 태깅. TCGPlayer→`en`, 네이버카페→`kr` |
| 2-3. 그레이딩 정규화 | 스크립트 + LLM 판단 | PSA/BGS 등급별 가격 정리. 네이버 카페 시세글에서 그레이딩 등급 파싱 (비정형 텍스트 → LLM) |
| 2-4. 가격 정규화 | 스크립트 | USD → KRW 환율 적용, 소스별 가격 병합. 동일 카드+에디션+등급 기준으로 병합 |
| 2-5. 이벤트 중복 제거 | 스크립트 + LLM 판단 | 동일 이벤트 다른 소스 → LLM이 유사도 판단하여 병합 |
| 2-6. 이벤트 분류 | LLM 판단 | 수집된 이벤트를 카테고리 분류 (오프라인 이벤트 / 카드샵 행사 / 콜라보 행사) |
| 2-7. 데이터 병합 | 스크립트 | 소스별 raw 데이터 → 통합 JSON 생성 |

**LLM 판단 영역 vs 코드 처리 영역**:

| LLM이 직접 수행 | 스크립트로 처리 |
|----------------|----------------|
| 미매핑 카드명의 한국어 대응 추론 | 매핑 테이블 기반 카드명 변환 |
| 네이버 카페 비정형 시세글에서 가격/등급 추출 | 환율 적용, 가격 단위 변환 |
| 이벤트 유사도 판단 및 병합 결정 | 에디션 태깅 (소스 기반 규칙) |
| 이벤트 카테고리 분류 | JSON 구조 병합, 중복 ID 제거 |
| 크롤링 실패 원인 분석 (구조 변경 감지) | PSA/BGS 등급 코드 정규화 |

**성공 기준**: 통합 JSON 파일 3개 생성 (`cards.json`, `lego.json`, `events.json`)
**검증 방법**: 스키마 검증 — 필수 필드 존재, 타입 체크, 가격 값 양수
**실패 처리**: 자동 재시도 (최대 2회) → 실패 시 이전 데이터 유지 + 로그

### Step 3: 검증

| 검증 항목 | 검증 유형 | 기준 |
|-----------|----------|------|
| JSON 스키마 유효성 | 스키마 검증 | 필수 필드 존재, 타입 일치 |
| 가격 이상치 탐지 | 규칙 기반 | 전일 대비 ±300% 이상 변동 → 경고 플래그 |
| 이벤트 날짜 유효성 | 규칙 기반 | 과거 30일 이전 이벤트 자동 아카이브 |
| 데이터 신선도 | 규칙 기반 | 마지막 수집 시각이 24시간 이내인지 |

**성공 기준**: 모든 검증 통과, 또는 경고 플래그가 붙은 채로 통과
**검증 방법**: 규칙 기반 (자동)
**실패 처리**: 이상치 → 경고 플래그 표시 후 배포 (데이터 삭제하지 않음). 스키마 실패 → 배포 중단 + 로그

### Step 4: 사이트 빌드

| 하위 작업 | 처리 방식 | 설명 |
|-----------|----------|------|
| 4-1. HTML 생성 | 스크립트 | 템플릿 + JSON 데이터 → 정적 HTML 생성 |
| 4-2. 자산 복사 | 스크립트 | CSS, JS, 이미지 → 빌드 디렉토리로 복사 |
| 4-3. 메타 정보 삽입 | 스크립트 | 마지막 업데이트 시각, 데이터 소스 상태 표시 |

**성공 기준**: `build/` 디렉토리에 `index.html` + 필요 자산 존재
**검증 방법**: 규칙 기반 — 파일 존재 여부 + HTML 유효성 기본 체크
**실패 처리**: 자동 재시도 (최대 1회) → 실패 시 배포 중단

### Step 5: 배포

| 하위 작업 | 처리 방식 | 설명 |
|-----------|----------|------|
| 5-1. gh-pages 푸시 | 스크립트 | `build/` 내용을 `gh-pages` 브랜치에 force push |
| 5-2. 배포 확인 | 스크립트 | GitHub Pages URL로 HTTP 200 확인 |

**성공 기준**: GitHub Pages URL 접근 시 200 응답 + 최신 업데이트 시각 표시
**검증 방법**: 규칙 기반 — HTTP 상태 코드 확인
**실패 처리**: 자동 재시도 (최대 2회) → 실패 시 GitHub Actions 실패 알림

## 2.3 상태 전이도

```
[대기] → [수집중] → [정제중] → [검증중] → [빌드중] → [배포중] → [완료]
                ↓          ↓          ↓          ↓          ↓
           [소스별      [재시도     [경고       [재시도     [재시도
            스킵]       또는        플래그      또는        또는
                        이전데이터   또는        배포중단]   실패알림]
                        유지]       배포중단]
```

## 2.4 네이버 카페 폴백 전략

네이버 카페 크롤링이 차단될 경우를 대비한 단계적 폴백. **이벤트/행사**와 **국내판 카드 시세** 두 가지 용도에 각각 적용.

**이벤트/행사 정보 폴백:**
```
1차: 네이버 카페 공개 게시판 크롤링 (로그인 없이 접근 가능한 게시글만)
  ↓ 실패 또는 데이터 없음
2차: 구글시트 수동 입력 → GitHub Actions에서 시트 데이터 fetch
  ↓ (이 단계는 항상 작동)
최종: 이벤트 탭에 "커뮤니티 정보는 수동 업데이트" 안내 표시
```

> ⚠️ 네이버 검색 API(2차 폴백)는 API 키 부재로 v1에서 제외. v2 확장 예정.
> 포켓몬코리아 SNS(인스타/X) 크롤링은 차단 강도 과다로 **제거**. 공식 웹사이트만 수집.

**국내판 카드 시세 폴백 (4단계)**:
```
1차: 번개장터 웹 크롤링 (키워드 검색 → 매물 가격 수집)
  ↓ 차단 (403/429/CAPTCHA)
2차: 당근마켓 웹 크롤링 (인기매물 + 검색 페이지)
  ↓ 차단
3차: 네이버 카페 시세글 크롤링 (공개 게시판 한정)
  ↓ 실패 또는 데이터 없음
4차: 구글시트에 KYH가 직접 국내 시세 입력 → GitHub Actions에서 fetch
  ↓ (이 단계는 항상 작동)
최종: 카드시세 탭에서 국내판 에디션에 "시세 미수집" 표시
```

> ⚠️ 당근/번개장터는 차단 가능성이 높으므로, 1~2차가 실패해도 3차(네이버 카페 공개 게시판)가 독립적으로 작동하도록 설계한다.
> 네이버 카페는 공개 게시판만 접근 가능하므로 로그인 필요 시세글은 수집 불가임을 인지한다.
> 차단이 3일 연속 발생하면 해당 소스를 자동으로 비활성화하고, 로그에 "수동 확인 필요" 알림을 남긴다.
