# 5. 프론트엔드 및 기술 스택

## 5.1 웹사이트 UI 구조

```
┌─────────────────────────────────────────────┐
│  🟡 포켓몬 굿즈 트래커                        │
│  마지막 업데이트: 2026.03.28 09:00            │
├──────────┬──────────┬──────────┬────────────┤
│ 💳 카드시세 │ 🎪 이벤트  │ 🧱 레고    │ ℹ️ 정보   │
├──────────┴──────────┴──────────┴────────────┤
│                                             │
│  [탭별 콘텐츠 영역]                           │
│                                             │
│  카드시세 탭:                                 │
│  ┌─ 필터 바 ──────────────────────────┐     │
│  │ 에디션: [전체] [한국판] [영어판]       │     │
│  │ 등급:  [미그레이딩] [PSA▼] [BGS▼]    │     │
│  └────────────────────────────────────┘     │
│  - 카드 목록 (이미지 + 이름 + 에디션 뱃지)     │
│  - 현재가 + 전일 대비 등락 (▲▼%)             │
│  - ⚠️ 이상치 배지 (전일대비 ±30% 이상 변동)   │
│  - 🕐 "최신 데이터: X일 전" (거래 드문 카드)  │
│  - 등급별 가격 비교 테이블 (펼치기/접기)       │
│                                             │
│  이벤트 탭:                                   │
│  - 카테고리 필터 (전체/오프라인/카드샵/콜라보)   │
│  - 이벤트 카드 리스트 (날짜순)                 │
│  - 진행중/예정/종료 구분                       │
│                                             │
│  레고 탭:                                     │
│  - 상품 목록 (이름 + 정가 + 중고시세)          │
│  - 프리미엄 비율 표시 (중고시세/정가)           │
│                                             │
│  정보 탭:                                     │
│  - 데이터 소스 목록 + 설명                     │
│  - 소스별 수집 상태 (source_status 기반)       │
│    예: TCGPlayer ✅ | 당근마켓 🚫 차단됨       │
│  - 마지막 업데이트 시각                        │
│                                             │
└─────────────────────────────────────────────┘
```

**UI 핵심 요구사항**:
- 모바일 반응형 (비개발자 대상, 모바일 비율 높음)
- **CSR (클라이언트 사이드 렌더링)**: index.html은 껍데기. 브라우저가 로드될 때 JS가 `/data/cards.json` 등을 fetch하여 화면을 구성. Jinja2 서버사이드 렌더링 사용 안 함.
- 탭 전환은 JS로 처리 (페이지 새로고침 없음)
- 헤더에 "마지막 업데이트: 2026.03.28 09:00" 상시 표시 (사용자가 직접 방문하여 최신 여부 확인)
- **에디션 필터**: 한국판/영어판 토글 (기본값: 전체)
- **그레이딩 필터**: 미그레이딩/PSA 등급별/BGS 등급별 드롭다운
- **등급별 가격 비교**: 카드 클릭 시 해당 카드의 모든 에디션×등급 가격을 테이블로 표시
- **이상치 경고 배지**: `warning_flags`에 항목이 있을 때 ⚠️ 배지 표시. 가격은 그대로 표시 (숨기지 않음). 임계값: 전일 대비 ±30% 이상 변동
- **"최신 데이터: X일 전" 인디케이터**: `last_seen_date`가 오늘로부터 7일 이상 된 카드에 표시. 콜라보 카드처럼 거래가 드문 카드에 유용
- **국내판 데이터 출처 표시**: 커뮤니티 기반 시세임을 UI에 명시 (⚠️ 비공식 시세, 판매 희망가)
- **국내 실거래 매물 표시**: 번개장터/당근마켓 매물 현황 (매물 수, 가격 범위) 별도 영역에 표시. 소스 상태(정상/차단) 뱃지 포함
- **소스 차단 안내**: 특정 소스가 `blocked` 상태일 때 UI에 "현재 수집 불가" 표시 (에러가 아닌 정보 수준)
- **정보 탭**: 데이터 소스 안내 + `cards.json`의 `source_status` 기반으로 소스별 수집 상태(✅정상/🚫차단/⚠️오류) 실시간 표시

## 5.2 기술 스택 요약

| 영역 | 기술 | 선택 근거 |
|------|------|----------|
| 크롤링 | Python (requests + BeautifulSoup) | 범용성, 라이브러리 풍부 |
| API 호출 | Python (requests) | eBay Browse API, 한국은행 환율 API 연동 |
| 데이터 처리 | Python (pandas 선택적) | JSON 처리, 데이터 정규화 |
| 사이트 빌드 | 빌드 스크립트 없음 (정적 파일 복사만) | **CSR 구조**로 HTML 사전 생성 불필요. `site/` 디렉토리를 그대로 gh-pages에 배포 |
| 프론트엔드 | Vanilla HTML/CSS/JS | GitHub Pages 호환, 빌드 도구 불필요. JS가 `/data/*.json`을 fetch하여 렌더링 |
| 환율 | 한국은행 공개 API (ECOS) | 무료, 정확한 KRW 시세. 일 1회 갱신 후 캐시 |
| LLM | Claude API (Anthropic) | 변경사항 있을 때만 호출. 카드명 매핑, 시세글 파싱, 이벤트 분류 |
| CI/CD | GitHub Actions | 무료, 리포지토리 내장 |
| 호스팅 | GitHub Pages (public 레포) | 무료, 정적 사이트 최적 |

## 5.3 데이터 전달 패턴

```
[수집 스크립트] → /data/raw/*.json (파일 기반)
                      ↓
[data-enrichment-agent] ← /data/raw/ 읽기 → /data/ 쓰기 (파일 기반)
                      ↓
[data-processor] → /data/cards.json, lego.json, events.json (파일 기반)
                      ↓
[validator] ← /data/*.json 읽기 → 검증 결과 로그 (파일 기반)
                      ↓
[site-builder] ← /data/*.json + /site/templates/ → /site/build/ (파일 기반)
```

모든 단계 간 데이터 전달은 **파일 기반**으로 통일한다. 중간 산출물은 `/data/raw/`와 `/data/`에 저장하고, 다음 단계에서 파일 경로를 참조한다.
